Endpoint to sign in a user with an email, password, and optionally a TOTP code. If the user requires 2FA, it will either: - Throw `TFARequired` if no TOTP code is provided, but the user has a TOTP secret. - Throw `TFARequiredOverEmail` if the user has no TOTP secret; in this case, the login should be done again on `POST /auth/signin/email` to request TFA over email. - May log in using the TOTP code, throw `InvalidTOTPCode` if the code is invalid.
